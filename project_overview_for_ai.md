# ゴール

[later-cli](https://github.com/kujirahand/later-cli)のデータを同期するためのAPIとWebサービス。

PHPでAPIとWebサービスを構築する。
Webサービスが/index.php、APIが/api.phpにアクセスする。

## 仕様

マルチユーザー対応。

全ユーザー情報を/data/users.dbに保存。`users.sql`からDBを作成する。

各ユーザーのデータを/data/users/user-0.db などのようにに保存。、10人ごとに異なるsqliteのDBに保存する

### ファイル構成

```text
later-api/
├── api.php --- APIのエントリポイント
├── config.inc.php --- 設定ファイル
├── config.tpl.php--- 設定ファイルのテンプレート
├── data/ --- データ保存用ディレクトリ
│   ├── .htaccess --- データディレクトリへのアクセスを禁止するためのファイル
│   ├── users.db
│   └── users/
│       ├── user-0.db
│       ├── user-1.db
│       └── ...
├── templates/ --- Webサービスのテンプレート
│   ├── index.html
│   └── ...
├── index.php
├── README.md
├── users.sql
```

`config.tpl.php`の内容を`config.inc.php`にコピーして、必要な値を設定して使う。

## Webサービス

メール認証を行って、ユーザーログインする。
ログインするたびにメール認証が必要。パスワードレスの仕様。
パスワード認証で、ログインしたら、APIキーの発行画面に遷移する。

### 新規ユーザー作成画面 - メール認証

新規ユーザー登録のメニューから、メールアドレスを入力して、認証メールを送信する。
メールに記載されたURLをクリックして、認証を完了する。
トークンは数字6文字とする。

メールの文面は、次の通り：

```text
件名: Later APIの認証メール

Later APIをご利用いただきありがとうございます。
以下のトークンをLater APIの認証ページに入力して、認証を完了してください。
---
123456
---
もし、このメールに心当たりがない場合は、削除してください。
申し訳ありませんが、このメールに返信しても対応できませんので、ご了承ください。
```

メール送信後の画面には、メールソフトの迷惑メールフォルダをご確認くださいと表示する。
メールの送信元情報などは、`config.inc.php`から読み取る。

APIキーを発行し、そのAPIキーを使って、データの同期を行う。
メールアドレスで認証する

### APIキーの発行

ログインしたら、ユーザーは、APIキーを好きな数だけ発行したり削除したりできる。APIキーは、`laterapi::{user_id}::{hash}{token}`の形式とする。hashは現在日時とemailと乱数を加えてsha256でハッシュ化してbase64にしたもの、tokenは、セキュリティ的にランダムな文字列16桁とする。

APIキーは、ユーザーごとに10個まで発行できるものとする。APIの発行日時と有効期限を保存する。APIキーの有効期限は、発行から1年/3年/5年/10年から選択するものとする。APIキーの有効期限が切れたら、APIキーは無効になるものとする。無効なAPIキーは、APIのリクエストを拒否するものとする。

## APIの仕様

APIは、HTTPリクエストでアクセスするものとする。APIのエンドポイントは、`/api.php`とする。APIのリクエストは、HTTPヘッダーにAPIキーを含めるものとする。APIのレスポンスは、JSON形式とする。全てのリクエストPOSTリクエストの元に行われる

認証は、bearerトークン方式で行うものとする。APIキーは、HTTPヘッダーの`Authorization`に`Bearer {API_KEY}`の形式で含めるものとする。APIキーが有効であれば、APIのリクエストを処理するものとする。APIキーが無効であれば、APIのリクエストを拒否するものとする。

### イベントの記録(post)

イベントの内容をDBに記録するメソッド。

- エントリポイント: POST /api.php?method=post

JSON形式で、次のようなリクエストを受け取る。

```json
{
    "events" : [
        {
            "event": "add",
            "guid": "123e4567-e89b-12d3-a456-426614174000",
            "timestamp": "2026-05-29 14:00:00",
            "task": "タスクの内容",
            "status": "done",
            "date": "2026-05-30 08:00:00"
        },
        {
            "event": "delete",
            "guid": "123e4567-e89b-12d3-a456-426614174000",
            "timestamp": "2026-05-29 14:00:00",
            "task": "タスクの内容",
            "status": "done",
            "date": "2026-05-30 08:00:00"
        }
    ]
}
```

上記のeventsの内容をユーザーのDBに保存する。APIキーが無効であれば、APIのリクエストを拒否する。

### イベントの取得(get)

イベントの内容をDBに記録するメソッド。

- エントリポイント: /api.php?method=get

JSON形式で、次のようなリクエストを受け取る。

```json
{
    "date_from": "2026-05-29 00:00:00",
    "date_to": "2026-05-30 23:59:59"
}
```

ユーザーのDBから、`date_from`から`date_to`までのイベントを取得して、JSON形式でレスポンスする。APIキーが無効であれば、APIのリクエストを拒否する。date_toが指定されない場合は、現在日時をdate_toとする。date_fromが指定されない場合は、date_toの7日前をdate_fromとする。

### API認証テスト(hello)

APIキーが有効であれば、APIのリクエストを処理するものとする。APIキーが無効であれば、APIのリクエストを拒否するものとする。

- エントリポイント: /api.php?method=hello

JSON形式で、次のようなリクエストを受け取る。

```json
{
    "message": "Hello, Later API!"
}
```

APIキーが有効であれば、次のようなJSON形式のレスポンスを返す。

```json
{
    "message": "Hello, Later API!"
}
```
